VZ: MusicBrainz — Show All Entity Data In A Consolidated View
=============================================================

PURPOSE
-------
Consolidates paginated and non-paginated MusicBrainz table lists (Events,
Recordings, Releases, Works, Artists, Labels, Release Groups, and more) into a
single scrollable, filterable, sortable view.

SUPPORTED PAGES
---------------
• Artist:   Release Groups, Releases, Recordings, Works, Events, Aliases,
            Relationships (including filtered link_type_id pages)
• Release Group, Release, Recording, Work, Label, Series, Place, Area,
  Instrument, Event pages — all supported sub-tabs
• Search result pages (all entity types)

GETTING STARTED
---------------
1. Navigate to any supported MusicBrainz entity page (e.g. an Artist's
   Recordings tab).
2. Click the "Show all …" action button in the controls bar to start fetching
   all pages from the MusicBrainz database.
3. A real-time progress bar (light-red → orange → light-green) tracks the
   fetch.  Click "Stop" at any time to abort; or press the prefix key then "o"
   while the Stop button is visible.
4. When complete, the consolidated table appears with filtering, sorting, and
   all UI features active.  The global filter is auto-focused so you can start
   typing immediately.

ACTION BUTTONS
--------------
• 💾 Save to Disk    — save the current dataset as a compressed .json.gz file
                       (~60–80% smaller than plain JSON); the filename encodes
                       page type, context detail, row count, and timestamp
• 📂 Load from Disk  — load a previously saved dataset via the three-phase
                       Load → Filter → Render dialog (see PRE-FILTER section)
• ↔️ Resize          — fit all columns to their content (toggle to restore)
• 📊 Statistics       — show a statistics panel (row/column counts, memory, …)
• 📏 Density         — choose Compact / Normal / Comfortable row spacing
• 👁️ Visible         — show/hide individual table columns; Alt-S / Alt-D to
                       select/deselect all; navigate with ↑/↓ / Tab / Shift-Tab
• 💾 Export          — export visible data as CSV, JSON, or Emacs Org-Mode
• 🎹                 — show keyboard shortcuts reference (or press ? / /)
• ⚙️                 — open the Settings Manager to configure all options
• ❓                 — show this help dialog

FILTERING
---------
• Global Filter — large input box at the top filters all columns at once
• Column Filters — per-column filter row below the table header
• Sub-table Filters — independent filter inputs above each sub-table on
  multi-table pages (e.g. Artist-Relationships)
• All filter inputs support plain text, case-sensitive (Cc checkbox),
  regexp (Rx checkbox), and exclude-matches (Ex checkbox) mode — tick Ex to
  hide rows that match instead of keeping them
• Filter highlighting works across inline HTML element boundaries (cross-tag
  matching) — search terms spanning <a>, <bdi>, or <span> elements highlight
  correctly in all three filter levels
• Ctrl-G  — focus global filter field (configurable)
• Ctrl-C  — focus first column filter; repeated presses cycle through tables
            (configurable)
• Ctrl-Shift-G — clear all active filters (configurable; also activates the
                "✕ Clear ALL filters" button)
• Shift-Esc    — clear all COLUMN filters only (global, works from anywhere)
• Escape  — first press clears the focused filter, second press removes focus
• 🎨 Toggle highlighting — toggle highlight colours on/off for active filters

FILTER INPUT STATES & BORDER COLORS
-------------------------------------
Every filter input transitions through three visual states:
• Idle (empty)  — configurable border color (global default: black,
                  sub-table default: #cccccc)
• Active (has text) — configurable border color (global default: orange,
                      sub-table default: green)
• Error (invalid RegExp when Rx is on) — shared configurable error color
  (default: dark red #cc0000, border width increases to 4 px for visibility);
  an "⚠ Invalid regexp: …" message is written to the filter status area
Both global and sub-table filter fields are resizable via a ⋮ drag handle
to their right; initial widths are configurable in ⚙️ Settings.

COLUMN FILTER FOCUS INDICATORS
--------------------------------
When a column filter field is focused (via Ctrl-C, Tab, or mouse click):
• A configurable search-icon prefix (default "🔍 ") is prepended to the field
  to make it immediately obvious which column is being edited.  The prefix is
  purely decorative: it is never included in the actual filter string.
• The field background turns light yellow (configurable colour
  "Column Filter Focus Background", default #fffde7).
• The prefix cannot be deleted or selected — Backspace, Delete, ArrowLeft,
  and Home are clamped so the cursor stays at or after the prefix.  Ctrl+A
  selects only the user-editable text after the prefix.
When the field loses focus:
• The prefix is always removed from the stored value.
• If the user entered a filter string the background switches to a persistent
  "active" colour (configurable "Column Filter Active Background", default
  #fff9c4), providing a visual overview of which columns have active filters.
• If the field was cleared the background is removed entirely.
Both colours and the prefix string are configurable in ⚙️ Settings →
🎨 FILTER HIGHLIGHT COLORS.

UNIQUE-VALUES COLUMN DROPDOWN  (📊 button in every column header)
------------------------------------------------------------------
Each column header contains a 📊 button that opens a scrollable panel listing
every distinct non-empty value currently visible in that column.

• Occurrence count badge — each entry is prefixed by a styled "(n)" badge
  showing how many visible rows contain that value.  All badges within one
  column share a uniform monospace width so the numbers right-align:
      (12) value A
    (5678) value B
   (21345) value C
  Badge text and background colors are configurable in ⚙️ Settings →
  🎨 FILTER HIGHLIGHT COLORS ("Unique-Values Count Badge" settings).
• Quick-filter bar — a "Quick filter…" input at the top of the panel is
  auto-focused when the panel opens.  Type to narrow the item list in real
  time; matching substrings are highlighted (configurable colors).  A ×
  button clears the filter.
• Selecting a value — click any item, or navigate with ↑/↓ / Home / End
  and press Enter, to write the value directly into the column's filter
  input and apply the filter immediately.
• Escape — first press clears the quickfilter (if non-empty); second press
  (or first press when quickfilter is already empty) closes the panel.
• The panel closes automatically when clicking outside it or when the page
  is scrolled (scrolling inside the panel itself does not close it).

PRE-FILTER (Load from Disk — three-phase dialog)
-------------------------------------------------
The Load-from-Disk dialog works as a sequential three-phase workflow:

  Phase 1 — Load:   Click "Load Data" (Alt+L) to pick a .json.gz or .json
                    file.  The entire file is read and parsed into memory.
                    A ✅ status line reports the row count and filename.

  Phase 2 — Filter: Enter an optional filter expression (plain text, RegExp,
                    or Exclude Matches) using the Cc / Rx / Ex toggles, then
                    click "Filter Data" (Alt+F).  Matching rows are counted
                    in memory — no DOM changes occur yet.  The count is shown
                    ("N of T rows match → will be rendered").

  Phase 3 — Render: Click "Render Data" (Alt+R) to apply the filter and
                    render the matched rows.  The dialog closes when done.

• The last N expressions (configurable via "Load Filter History Limit") are
  saved and accessible via the ▼ dropdown toggle next to the filter field.
• Pre-filtered rows are highlighted with 🎨; toggle highlighting with the
  dynamic prefilter button that appears in the filter bar.
• Escape dismisses the dialog at any phase.

SORTING
-------
• Click any column header (⇅) to sort ascending; click again for descending;
  click the ⇅ icon again to restore original order
• Visual indicator changes to ▲ / ▼ on the active sort column
• Multi-column sort: Ctrl+Click ▲ or ▼ to add a column to the sort chain;
  superscript numbers (¹²³…) show each column's priority; plain click returns
  to single-column mode
• Large tables use an async chunked merge-sort; a progress indicator is shown
  when the row count exceeds the configurable sort-progress threshold

COLLAPSING / EXPANDING
-----------------------
• Click an h2 header to collapse/expand its table section
• Ctrl-Click an h2 to toggle all h2 headers simultaneously (or use Ctrl-2)
• Click an h3 header to collapse/expand its type group
• Ctrl-Click an h3 to toggle all h3 headers simultaneously (or use Ctrl-3)
• Show All / Hide All links available per section

SAVE & LOAD (Offline Cache)
----------------------------
• Datasets are saved as gzip-compressed JSON (.json.gz) — ~60–80% smaller
  than plain JSON.
• Generated filenames follow: MB-<pageType>[-<detail>-]<rowCount>-<timestamp>.json.gz
  The optional <detail> slug is derived from the active button label (on
  multi-button pages) or from the relationship h2 heading (on link_type_id
  filtered pages), making saved files immediately distinguishable.
• When loading, enter an optional pre-filter expression (plain text, RegExp,
  or Exclude-Matches mode) to import only the rows you need.
• Load filter history (last N expressions, configurable) is accessible via the
  ▼ dropdown in the Load dialog.
• Pre-filtered rows are highlighted with 🎨; toggle highlighting with the
  dynamic prefilter button.

COLUMN MANAGEMENT
-----------------
• 👁️ Visible menu — check/uncheck any column; Select All (Alt-S) /
  Deselect All (Alt-D); button turns red when any column is hidden
• Manual resize — drag column-header edges with the mouse at any time
• Resize — calculates optimal widths for content including images/icons;
  acts as a toggle (restores original widths on second click)
• Synthetic columns — some page types add computed columns alongside the raw
  data:
    — "Country" + "Date"        split from the combined "Country/Date" cell on
                                release and label pages
    — "Place" + "Area" + "Country"  split from the "Location" cell on event
                                    and area pages
    — "MB-Area" + "Country"     split from the "Area" cell on label, area,
                                and recording pages
    — "Format Types"            strips quantity prefixes (e.g. "8×") from
                                the Format column on release-group, label, and
                                series pages
    — "MB-Name" + "Comment"     extracted from the main entity name cell on
                                pages that expose the raw MusicBrainz entity
                                name alongside its disambiguation comment

H1 ALIAS LINE
-------------
On Release Group and Event pages MusicBrainz sometimes places the primary alias
inside the h1 title.  The script relocates that alias into a dedicated italic
line immediately below the title: «Primary alias: …», keeping the title line
and action buttons uncluttered.  Extra show-title text (when present alongside
the alias in the same span) is preserved as a new comment span in the h1.

DENSITY CONTROL
---------------
  Compact     — tighter padding, smaller font, more rows visible
  Normal      — default MusicBrainz-like spacing
  Comfortable — larger padding for easier reading
  Navigate with ↑/↓ in the menu for an immediate live preview; Enter to apply

EXPORT
------
  CSV            — comma-separated, compatible with Excel / Google Sheets
  JSON           — structured JSON array of visible rows
  Emacs Org-Mode — pipe-delimited Org table format

PROGRESS BAR
------------
• The inline progress bar in the controls line transitions from light red →
  light orange → light green as pages are fetched (0 % / 50 % / 100 %).
• The action button for the current load changes colour in sync.
• A time estimate ("est. X.Xs left") is shown inside the bar.

STATISTICS PANEL (📊 or Ctrl+I)
--------------------------------
• Shows: sub-table count, total rows, visible rows, column count, multi-sort
  chain, estimated memory, and per-category row counts on multi-table pages.

SETTINGS (⚙️ or Ctrl+,)

the top filters the settings list in real time, highlighting matching text in
labels and descriptions.  Major setting groups:

• 🛠️ GENERIC SETTINGS

      pages like Artist-Relationships; enable to open them in a new tab)

    — Collapsible sidebar

• 🧮 OPTIONAL COLUMN REMOVAL
    — Remove Tagger, Rating, Relationships, Performance, or "Release events"
      columns from the final rendered tables

• 🎹 KEYBOARD SHORTCUTS
    — Enable / disable keyboard shortcuts and prefix-mode tooltip overlay
    — Configure the prefix key (default: Ctrl+M)
    — 20 configurable direct-chord shortcuts:
         Global: Save, Load, Resize, Visible, Density, Statistics, Export,
                 Settings, Global Filter, Column Filter, Clear Filters,
                 Toggle h2, Toggle h3, Unicode Character Picker
         Column-filter-focused: Sort Ascending, Sort Descending, Unsort,
                 Toggle Multi-Row Collapse, Unique-Values Dropdown,
                 Toggle CAA/EAA Cover Art

• 🎨 FILTER HIGHLIGHT COLORS

    — Column filter focus background and active background colors
    — Filter focus prefix string (default "🔍 ")
    — Global filter border colors: idle, active, error

SETTINGS DIALOG SAFETY GUARDS
------------------------------
• If you close the dialog (via the CLOSE link, Escape, or clicking outside)
  while there are unsaved changes a confirmation dialog ("Unsaved Changes")
  appears.  Click OK to discard the changes, or Cancel to return to the dialog.
• Clicking RESET always opens a confirmation dialog ("Reset All Settings to
  Defaults") before any defaults are applied. Click OK to proceed, or Cancel
  to abort.

CHANGELOG DIALOG (📜 ChangeLog in the Tampermonkey menu)
----------------------------------------------------------
• The changelog is loaded from GitHub (with a 1-hour GM cache) at startup.
• A 🔍 search field in the dialog header filters entries in real time;
  matching text is highlighted in version, date, section, and item text.
  A ✕ button clears the search field.
• A "(N filtered)" count is shown in the meta line when a search is active.
• When you open the Changelog dialog the console shows cache-status info:
  entry count, source (📦 cache / 🌐 fresh fetch / static), and cache age.
• A "🔄 Force refresh" link in the upper-right corner (left of CLOSE) bypasses
  the cache, downloads the latest changelog from GitHub, and re-renders the
  table in place — without closing the dialog.  The current search filter is
  preserved after a force refresh.
• Force-refresh success / failure is reported in the browser console and
  reflected briefly in the link label (✅ Refreshed / ⚠️ Refresh failed).

APPLICATION HELP DIALOG (❓ button or prefix key + H)
-------------------------------------------------------
• The help text is fetched from GitHub (with a 1-hour GM cache) when the
  dialog opens.  The console reports the URL, source (📦 cache / 🌐 fresh),
  and byte size.
• A "🔄 Force refresh" link in the title bar (left of ✕) downloads the latest
  help text from GitHub and re-renders the content in place.
• While a force refresh is in progress the link shows "⏳ Refreshing…", then
  briefly "✅ Refreshed" on success before returning to its default label.

LIBRARY SETTINGS (🔧 Library Settings in the Tampermonkey menu)
----------------------------------------------------------------
• A small settings editor for the shared VZ_MBLibrary, accessible from the
  Tampermonkey script menu.
• Currently exposes: content font size and content font family, which apply
  to the Changelog dialog and (when read by the consumer script) to the
  App Help dialog.

SIDEBAR
-------
• The page sidebar can be optionally collapsed (experimental setting) to give
  more horizontal space to the data table.
• Sticky table headers remain visible while scrolling through large tables
  (can be disabled in Settings).

MULTI-TABLE PAGES
-----------------
• On pages like Artist-Relationships or Release-Group releases the data is
  split into independent sub-tables per category, each with its own filter
  (including resizable input, idle/active/error border states, and regexp
  error reporting to the status span), sort chain, clear button, and optional
  "Show all N" overflow expander.
• A single sub-table is auto-expanded on load.
• Ctrl+Click sort applies independently per sub-table.
• Overflow sub-tables exceeding 100 rows can optionally be opened in a new
  browser tab (configurable in ⚙️ Settings → 🛠️ Generic Settings).

PAGE RELOAD NOTICE
------------------
When MusicBrainz applies a URL filter that would interfere with pagination,
the script automatically reloads the page to strip the filter.  After reload
an ⚠️ Page Reloaded notice appears and automatically re-clicks the "Show all"
button after 2 seconds (or immediately when you click OK / press Enter).
Press Escape on that notice to cancel the auto-action.
